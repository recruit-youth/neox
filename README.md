# NEOX Recruit — 採用ページ

**株式会社ネオックス** の採用専用 1ページサイト。  
unidge.co.jp のデザインテイストを参考にしたエディトリアル・ミニマルデザイン。  
求職者が「ここ気になる／話を聞いてみたい」と感じ、カジュアル面談・エントリー・LINE相談へ誘導することを目的とした採用ランディングページ。

---

## デザイン方針

| 項目 | 内容 |
|---|---|
| カラー | オフホワイト `#F5F2ED` × チャコール `#111111` × ゴールド `#C8A96E` |
| タイポ見出し | Shippori Mincho（和文） + Cormorant Garamond（欧文・装飾） |
| タイポ本文 | Noto Sans JP weight:300 |
| 基調 | 余白を大きく取ったエディトリアル構成、文字をデザインとして使う |
| トーン | 静かな高級感・誠実・共感ベース。にぎやかな装飾なし |

---

## 実装済み機能

### ローダー
- 初回アクセス時に NEOX ロゴ + プログレスライン表示（1.8秒）
- フェード + スケールインでヒーローへ遷移

### アニメーション
- ヒーロー大型タイポ：1行ずつ下からスライドイン（時差stagger）
- スクロール起動リビール（`IntersectionObserver` + stagger delay）
- 数字セクションのカウントアップ（ease-out-quart）
- ヒーロー `NEOX` 背景テキストの微細パララックス
- Vision カードのホバーグループ調光エフェクト
- ページ上部の読書進捗バー（ゴールドグラデーション）
- `prefers-reduced-motion` 対応（全アニメを無効化）

### ナビゲーション
- スクロール60px以上でフロスト（`backdrop-filter`）背景付き固定ナビに変化
- スムーズスクロール（アンカーリンク）

### エントリーフォーム
- バリデーション（名前・メール・種別）
- 送信データを `tables/entries` REST APIに保存
- 送信成功時にフォームを成功メッセージに切替

## データモデル

### `entries` テーブル

| フィールド | 型 | 説明 |
|---|---|---|
| id | text | 自動生成ID |
| name | text | 応募者氏名 |
| email | text | メールアドレス |
| phone | text | 電話番号（任意） |
| contact_type | text | casual / entry / line |
| message | rich_text | メッセージ（任意） |
| submitted_at | datetime | 送信日時 |

**APIエンドポイント：** `tables/entries`

## 推奨次ステップ

1. **実写社員写真の差し替え** — SVGプレースホルダーを実写に置換（最重要）
2. **LINE公式アカウントURLの設定** — `lin.ee/neox-recruit` を実URLに変更
3. **メール通知連携** — バックエンドAPIでフォーム送信時に採用担当へ通知
4. **OGP / Twitter Card 設定** — SNSシェア時のサムネイル最適化
5. **Google Analytics** — 求職者のスクロール深度・CTA クリック計測
6. **会社の実コピー確認** — 給与・所在地・採用条件を実情報に差し替え
